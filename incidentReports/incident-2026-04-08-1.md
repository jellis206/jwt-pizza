# Incident: 2026-04-08

## Summary

Between the hours of {START_TIME} and {END_TIME} on April 8, 2026, pizza order fulfillment experienced failures due to a chaos injection event from the JWT Pizza Factory. The event was detected by Grafana alerting rules monitoring pizza order failure rates. The incident affected all users attempting to place pizza orders during the chaos window.

## Detection

This incident was detected when the "Pizza Order Failures Detected" alert rule was triggered in Grafana, which monitors the `pizza_failures_total` metric for any increase. The alert fired within minutes of the first failed order.

## Impact

For approximately {DURATION} between {START_TIME} UTC and {END_TIME} UTC on 04/08/2026, pizza orders submitted to the JWT Pizza Factory were failing. Users who attempted to order pizzas received 500 error responses. Menu browsing and authentication remained unaffected.

## Response

After the alert was triggered, the on-call engineer reviewed the Grafana dashboard and observed a spike in `pizza_failures_total` along with error log entries from the factory endpoint. The factory response body contained a `reportUrl` field pointing to the chaos resolution endpoint.

## Root cause

The JWT Pizza Factory injected a chaos event that caused the factory's `/api/order` endpoint to return error responses. This is the expected behavior of the chaos testing system - the factory simulates an outage to validate our alerting and incident response capabilities.

## Resolution

The incident was resolved by accessing the `reportUrl` (also labeled `followLinkToEndChaos`) returned in the factory's error response. Visiting this URL disabled the chaos injection, and subsequent pizza orders succeeded normally.

## Prevention

- Ensure continuous traffic simulation is running so that chaos events are detected promptly via order failure metrics
- Maintain alert rules with appropriate thresholds for pizza failure rate, latency, and error detection
- Document the chaos resolution process (follow the `reportUrl`) for future incidents

## Action items

1. Verify all alert rules are correctly configured and firing as expected
2. Keep continuous traffic simulation running during business hours to ensure timely detection
3. Add a dashboard annotation for chaos testing events to distinguish from real incidents
